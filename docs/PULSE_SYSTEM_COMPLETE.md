## PULSE STATUS LEVELS

- **HEALTHY**: System is active and updated within last hour
- **WARNING**: System hasn't updated in 1-24 hours
- **CRITICAL**: System has issues
- **OFFLINE**: System hasn't updated in 24+ hours

## API ENDPOINTS

### Status & Overview
- `GET /api/pulse/status` - Pulse API status
- `GET /api/pulse/overview` - Complete pulse overview

### Systems
- `GET /api/pulse/systems` - Get all systems
- `GET /api/pulse/systems/{system_id}` - Get specific system pulse
- `POST /api/pulse/systems/{system_id}/update` - Update system pulse

### Domains
- `GET /api/pulse/domains` - Get all domains
- `GET /api/pulse/domains/{domain}` - Get specific domain pulse
- `POST /api/pulse/domains/{domain}/update` - Update domain pulse

### Integration
- `GET /api/pulse/integration-map` - Get integration map

