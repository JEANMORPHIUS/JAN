## Purpose
Defines operational rules, workflows, and standards for Siyem Media entity within the JAN MUHARREM ecosystem, aligned with actual services (entity_router.py, project_manager.py, campaign_exporter.py, etc.).

## Operational Principles

### Core Alignment
- All operations must align with telos.md principles
- Media content must reflect essence.md identity
- Operations follow jan_engine.prompt execution logic
- Governance from Siyem.org must be respected

### Quality Standards
- High-quality media outputs
- Consistent with system standards
- Alignment with creative identity
- Professional production values

## Media Management Rules

### Content Organization
- Clear folder structure and naming conventions
- Consistent metadata and tagging
- Version control for media assets
- Proper categorization and cataloging

### Asset Management
- Secure storage of media assets
- Backup and recovery procedures
- Access control for sensitive content
- Asset lifecycle management

### Production Workflows
- Standardized production processes
- Quality checkpoints at key stages
- Clear approval workflows
- Documentation of production decisions

## Operational Workflows

### Media Production
1. Receive production request or initiate project
2. Check alignment with telos.md and essence.md
3. Apply relevant style modules
4. Coordinate with creative entities as needed
5. Execute production workflow
6. Quality assurance check
7. Finalize and deliver media asset

### Content Management
1. Receive or create media content
2. Organize according to structure
3. Apply metadata and tagging
4. Store in appropriate location
5. Update catalog and documentation
6. Maintain version control

### Publishing Workflow
1. Review content for quality and compliance
2. Verify licensing status (licensing_pipeline.md)
3. Obtain publishing approval (publishing_oversight.md)
4. Set up distribution channels
5. Execute publication
6. Monitor post-publication performance
7. Complete audit trail

### Quality Assurance
1. Review media content for quality
2. Verify alignment with core principles
3. Check compliance with style modules
4. Validate against operational standards
5. Document quality status
6. Approve or request revisions
