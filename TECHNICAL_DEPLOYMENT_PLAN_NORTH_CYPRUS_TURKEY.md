### 8.4 DATABASE SCHEMA

**PostgreSQL Tables:**

```sql
-- Users table
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(50) NOT NULL, -- student, teacher, admin
  school_id UUID REFERENCES schools(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Schools table
CREATE TABLE schools (
  id UUID PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  country VARCHAR(100) NOT NULL, -- north_cyprus, turkey
  license_type VARCHAR(50), -- school, teacher, student
  license_expires_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Enrollments table
CREATE TABLE enrollments (
  id UUID PRIMARY KEY,
  student_id UUID REFERENCES users(id),
  lesson_id VARCHAR(255) NOT NULL,
  status VARCHAR(50), -- not_started, in_progress, completed
  progress INTEGER DEFAULT 0, -- percentage
  completed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Assessments table
CREATE TABLE assessments (
  id UUID PRIMARY KEY,
  student_id UUID REFERENCES users(id),
  lesson_id VARCHAR(255) NOT NULL,
  score INTEGER, -- percentage
  submitted_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Payments table
CREATE TABLE payments (
  id UUID PRIMARY KEY,
  school_id UUID REFERENCES schools(id),
  amount DECIMAL(10,2) NOT NULL,
  currency VARCHAR(10) NOT NULL, -- TRY, EUR, USD
  status VARCHAR(50), -- pending, completed, failed
  stripe_payment_id VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 8.5 SECURITY IMPLEMENTATION

**Authentication:**
- JWT tokens (15-minute access token, 7-day refresh token)
- Secure password hashing (bcrypt, 12 rounds)
- Rate limiting (max 5 login attempts per 15 minutes)
- Session management (secure cookies, httpOnly)

**Authorization:**
- Role-based access control (student, teacher, admin, super_admin)
- Resource-level permissions (students can only access their own data)
- API key authentication (for partner integrations)

**Data Protection:**
- Encryption at rest (AWS RDS encryption, S3 encryption)
- Encryption in transit (TLS 1.3, HTTPS only)
- PII data masking (in logs, analytics)
- Secure file uploads (virus scanning, file type validation)

**Privacy Compliance:**
- GDPR compliance (EU/North Cyprus)
- KVKK compliance (Turkey data protection law)
- Student data privacy (parent consent, data minimization)
- Right to deletion (GDPR Article 17)
- Data portability (export user data on request)

**Monitoring & Incident Response:**
- Real-time security monitoring (intrusion detection)
- Automated threat alerts
- Incident response plan (24-hour breach notification)
- Regular security audits (quarterly penetration testing)
- Bug bounty program (responsible disclosure)
